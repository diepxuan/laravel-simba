## 2026-02-21 - Git Workflow Bài học Quan trọng

### 📚 Bài học Git Workflow (3 lần lỗi liên tiếp)
1. **PR #2**: Merge thiếu commit Vietnamese docs → Website có menu English, không có tiếng Việt
2. **PR #3**: Merge thiếu commit logo → Website có tiếng Việt, không có logo
3. **PR #7**: Convert HTML → Markdown gây lỗi rendering → Website hiển thị linh tinh

### 🎯 Root Cause
- **GitHub PR mechanics**: PR là SNAPSHOT của branch tại thời điểm mở
- **Chỉ merge commits có sẵn** khi PR mở, bỏ qua commits thêm sau
- **Lỗi pattern**: Tạo PR → Thêm commits → Merge → Missing commits

### ✅ Quy trình mới (BẮT BUỘC)
1. **KHÔNG BAO GIỜ push vào PR đang mở**
2. **LUÔN LUÔN tạo branch mới** cho mỗi set of changes
3. **LUÔN LUÔN tạo PR mới** cho mỗi branch
4. **KIỂM TRA kỹ** commits trong PR trước khi merge
5. **VERIFY** tất cả changes được merge thành công

### 📝 PR Creation Checklist (BẮT BUỘC)
#### Trước khi tạo PR:
- [ ] Tất cả changes đã commit
- [ ] Không có uncommitted changes
- [ ] Branch name đúng convention (feat/xxx, fix/xxx, docs/xxx)
- [ ] Commit message descriptive
- [ ] PR description đầy đủ

#### Trước khi merge PR:
- [ ] Verify tất cả commits trong PR
- [ ] Check diff trên GitHub
- [ ] Test changes locally (nếu cần)
- [ ] Đảm bảo không missing commits

#### Sau khi merge PR:
- [ ] Check website thực tế
- [ ] Verify tất cả changes được apply
- [ ] Log bài học nếu có lỗi
